Presentación

Características de argenmap.jquery

argenmap.jquery es un plugin para jQuery que permite combinar los mapas web del IGN con la API de Google Maps con el fin de obtener beneficio de los datos geoespaciales oficiales del IGN sobre las imágenes satelitales de google.

Multi instancia

Podés usar varios mapas en la misma página.

Encadenable

Podés utilizar argenmap.jquery como cualquier plugin de jQuery. 

$('#prueba').show().argenmap().css('border', '1px solid #000000');

jQuery completo

Con este body html... 

<body>
<div id="prueba1" class="mapa arriba"></div>
<div id="prueba2" class="mapa centro"></div>
<div id="prueba3" class="mapa abajo"></div>
</body>

... este código primero inicializa todos los mapas, y luego agrega un marcador en los dos primeros utilizando 2 selectores diferentes. 

$('.mapa').argenmap({
	accion: 'init',
	opciones:{
		center:[-34, -59],
		zoom:2,
		mapTypeControl: true,
		mapTypeControlOptions: {
			style: google.maps.MapTypeControlStyle.DROPDOWN_MENU
		},
		navigationControl: true,
		scrollwheel: true,
		streetViewControl: true
	}
});
 
$('#prueba2').argenmap({
	accion:'agregarMarcador',
	latLng:[-34.001, -59.03]
});
 
$('.mapa.arriba').argenmap({
	accion:'agregarMarcador',
	latLng:[-34.02, -59.01]
})

Acceso completo a la API de Google Maps

Aunque argenmap simplifica el uso de mapas en web con datos oficiales, también permite el acceso a la API de Google Maps nativamente.
Este ejemplo muestra cómo agregar un marcador utilizando un ícono desde un sprite 

					
$('#prueba').argenmap({
	accion: 'agregarMarcador',
	direccion: "Av. Cabildo 381",
	marker: {
		options:{
			icon: new google.maps.MarkerImage("ui-icons_222222_256x240.png", new google.maps.Size(16, 16), new google.maps.Point((14.5*1), (14.5*10)))
		}
	},
	map: {
		center: true,
		zoom: 14,
		mapTypeId: google.maps.MapTypeId.TERRAIN
	}
});

Contextual

El elemento div que invoca a argenmap es el contexto this de las funciones de callback y de eventos. 

					
$('#prueba').argenmap({
	accion: 'agregarMarcador',
	latLng: [-33, 151],
	map: {
		center: true,
		zoom: 8
	},
	callback: function() {
		$(this).css('border', '1px solid red');
	}
});

Formato de los datos

Simplificamos algunos formatos de la API de Google Maps con argenmap.jquery (pero igual se pueden utilizar los objetos originales de GMaps API).

latlng

Podés usar el objeto google.maps.LatLng pero de manera simplificada:

  • {lat:float, lng:float} : objeto de 2 números decimales
  • [lat, lng] array de 2 números decimales

LatLngBounds

El objeto google.maps.LatLngBounds lo simplificamos así:

  • {ne:LatLng, sw:LatLng} : objeto de 2 LatLng
  • [ne, sw] : array de 2 LatLng
  • {n:float, e:float, s:float, w:float} : objeto de 4 números decimales
  • [n, e, s, o] : array de 4 números decimales
Podés un ejemplo combinado acerca del uso de LatLngBounds en la función agregarRectangulo

Conversión de direcciones a coordenadas

La mayoría de las funciones resuelven direcciones automáticamente por medio del parámetro direccion en lugar de utilizar latLng.

Por ejemplo, la función agregarMarcador utiliza coordenadas (latLng) para posicionar un nuervo marcador. Podés usar una dirección en lugar de sus coordenadas. La dirección se transforma automáticamente a coordenadas.

Tené en cuenta que la conversión de direcciones a coordenadas es asincrónica por el intercambio entre tu browser y el servicio de Google Maps.

direccion

direccion puede ser:

  • string: la dirección como texto
  • GeocoderRequest: un objeto de google completo

Parámetros de argenmap

Para su uso principal, argenmap toma objetos como parámetros. Cada objeto es una acción a ejecutar. Estas acciones son ejecutadas según su orden en la lista de parámetros, una por vez, cuando la acción anterior haya terminado.

Este ejemplo primero inicializa el mapa, y luego agrega dos marcadores.

$('#ejemplo').argenmap({
	accion:'init',
	opciones: {
		center:[-34, -59],
		zoom: 5
	}
}, {
	accion: 'agregarMarcador',
	latLng:[-34, -59]
}, { 
	accion: 'agregarMarcador',
	latLng:[-34.2, -59.2]
});

El objeto de acción usa estos parámetros

  • accionstringnombre de la función a utilizar
  • opcionesobjetoopciones a aplicar al objeto que se está creando (marcador, overlay ...)

Con algunas acciones específicas se puede incrustar otros parámetros que no están en esta lista

API Funciones de argenmap

El valor de accion puede ser

  • agregarMarcadorcrea y agrega un google.maps.Marker
  • agregarMarcadorescrea y agrega varios google.maps.Marker
  • getdevuelve el primer / último o todos los objetos Google Maps creados (map, marker, ...)
  • initInicia el mapa
  • limpiarLimpia los objetos creados sobre el mapa

Callback

Las funciones de callback reciben un parámetro que es el resultado de la acción ejecutada y depende de la acción.

  • agregarMarcador manda un google.maps.Marker
  • agregarCirculo manda un google.maps.Circle
  • geoLatLng manda un google.maps.LatLng
  • ...

En este ejemplo podés ver dos cosas importantes de argenmap:
en la función de callback $(this) equivale a $('#prueba').
setCenter es una llamada a la función de google.maps.Map (setCenter no es una función de argenmap), por eso el argumento a esta función está en la array args. 

$('#prueba').argenmap({
	accion : 'geoLatLng',
	callback : function(latLng) {
		if (latLng) {
			$(this).argenmap({accion: 'setCenter', args:[ latLng ]});
		} else {
			alert('No se pudo obtener la geolocalización');
		}
	}
});

Eventos

Los eventos que quieras enganchar a un mapa se guardan en eventos y unicos. Todos usan este prototipo: function (origen , evento , data, )

  • origen depende de la acción (agregarMarcador manda un google.maps.Marker, ...)
  • evento es el evento javascript q disparó esta función
  • data es lo que el usuario ingresa y puede ser cualquier cosa (string, array, objeto, booleano, ...)

En este ejemplo, usamos una string como data para cada marcador y se muestra cuando el evento click se dispara en estos marcadores

$('#prueba').argenmap({
	accion:'init',
	opciones: {
		center:[-34,-59],
		zoom: 5
	}
}, {
	accion: 'agregarMarcadores',
	markers: [
		{lat: -25.695278, lng: -54.436667, data:'Cataratas del Iguazú!'},
		{lat: -34.05, lng: -59.2, data:'Atucha: central eléctrica'},
		{lat: -27.31,  lng: -66.608617, data:'Minera Lumbrera: buena mina'}
	],
	marker: {
		opciones: {
			draggable: false
		},
		events: {
			click: function(marker, event, data) {
				alert(data);
			}
		}
	}
});

Funciones de Google Maps

Como argenmap maneja objetos puros de Google Maps API, es posible utilizar directamente las funciones nativas de google.maps a través de la API de argenmap.jquery
Pero como las funciones son llamadas directamente, los beneficios de la API de argenmap.jquery no están disponibles para este uso (Me refiero a LatLng simplificado resolución de direcciones, ...).

Los parámetros de las funciones de google se guardan en una array de parámetros para ser aplicados a la función
Si el contexto de la función no es el google.maps.Map, el objeto para utilizar como contexto (this) necesita ser pasado como propiedad target.
Las funciones acá tienen que ser llamadas con mayúsculas y minúsculas de acuerdo a la API de Google Maps.

Este ejemplo ejecuta map.setCenter con el resultado de una resolución de dirección. No se especifica la propiedad target porque el map es el destinatario de setCenter. 

$('#prueba').argenmap({
	accion: 'getLatLng',
	direccion: '2 bis rue saint antoine, eguilles',
	callback: function(resultado){
		if (result){
			$(this).argenmap( {accion: 'setCenter', args:[ resultado[0].geometry.location ]} );
		} else {
			alert('Mala dirección !');
		}
	}
});

No recomendamos este tipo de uso porque es medio oscuro.
Es preferible manejar el objeto de la API de Google Maps directamente.

El mismo ejemplo que antes, pero obteniendo (acción get)el objeto puro google.maps.Map

$('#prueba').argenmap({
	accion: 'getLatLng',
	direccion: '2 bis rue saint antoine, eguilles',
	callback: function(resultado) {
		if (result) {
			$(this).argenmap('get').setCenter(resultado[0].geometry.location);
		} else {
			alert('Mala dirección 2!');
		}
	}
});
