Características de argenmap.jquery

argenmap.jquery es un plugin para jQuery que permite combinar los mapas web del IGN y OpenStreetMap con Google Maps API con el fin de obtener beneficio de los datos geoespaciales oficiales del IGN sobre las imágenes satelitales de google.

Multi instancia

Podés usar varios mapas en la misma página.

Encadenable

Podés utilizar argenmap.jquery como cualquier plugin de jQuery.

$('#mapa').show().argenmap().css('border', '1px solid #000000');

Acceso completo a la API de Google Maps

Aunque argenmap simplifica el uso de mapas en web con datos oficiales, también permite el acceso a la API de Google Maps nativamente.
Este ejemplo muestra cómo centrar el mapa en la latitud -34 y longitud -59 accediendo al objeto google.maps.Map que utiliza argenmap.jquery internamente. 

					
$('#mapa').data('gmap').setCenter( new google.maps.LatLng(-34,-59) );

API versión 1.2 NUEVA API. válida desde el 22/11/2012

Lista rápida de las funciones del plugin

Inicialización

  • argenmap()

Vista del mapa

  • centro()
  • zoom()

Capas

  • capaBase()
  • agregarCapaWMS()
  • agregarCapaKML()

Marcadores

  • agregarMarcador()
  • agregarMarcadores()

$(selector).argenmap()

Inicializa un mapa simple sobre un elemento HTML. Preferentemente divs con un alto definido en píxels.

$(selector).centro( lat, lng )

Luego de instanciar un mapa, puede usarse .centro(lat,lng) para cambiar el centro del mapa.

lat (requerido): número indicando los grados decimales de latitud

lng (requerido): número indicando los grados decimales de longitud

Si llamás a centro sin parámetros, devuelve el centro actual. Una array del tipo [lat,lng].

$(selector).zoom(zoom)

Aumenta o disminuye la escala del mapa.

zoom (requerido): 1 - 22 número indicando el nivel de zoom al que se quiere visualizar el mapa.

Si llamás a zoom() sin parámetros, devuelve el nivel de zoom actual

$(selector).capaBase(nombre)

Cambia la capa base actualmente visible en el mapa por la capa especificada por el parámetro nombre. Si no se le pasan parámetros devuelve el nombre de la capa base actualmente visible.

nombreEl nombre de una de las capas estándar (satellite o IGN) o el nombre de una capa superpuesta previamente con agregarCapaBaseWMS().

$(selector).agregarCapaWMS(opciones)

Superpone una capa WMS sobre las capas base.

Las opciones que pueden pasarse son los parámetros que pueden pasarse a cualquier WMS:

url
cadena de texto indicando la url del servidor WMS. Ejemplo: http://mapa.ign.gob.ar/wms?.
La URL debe finalizar con el signo ? o con el signo &.
capas
cadena de texto con los nombres (el parámetro LAYERS de WMS) de las capas que se quieren pedir al servidor WMS, separadas por coma. Ejemplo: "departamentos, rios, ciudades"

Opcionales
 
nombre
cadena de texto indicando el nombre internoque tendrá la capa. Recordá que es importante definir el nombre y que sea único para todas las capas que superpongas.
formato
valor por defecto: "image/png" Cadena de texto indicando un formato MIME de archivo: "image/jpeg", "image/jpg", "image/png", "image/png8", "image/gif".

El servicio WMS tiene que estar configurado para poder generar las imágenes con el sistema de referencia EPSG:3857 que es el código que define la proyección de OpenStreetMap, Google Maps, Bing Maps, ESRI WEB Maps y demás mapas web populares.

$(selector).agregarCapaKML(opciones)

Superpone una capa que se importa desde un archivo .kml que esté online accesible por URL.

Sus opciones son:
url
cadena de texto con la URL del KML.

Opcionales
 
nombre
cadena de texto indicando el nombre que tendrá la capa. Recordá que es importante definir el nombre e intentar que sea único.

El archivo KML tiene que estar accesible públicamente. Este requerimiento de argenmap.jquery va a cambiar en la versión 2.0.

$(selector).agregarMarcador([opciones])

Trabajar con marcadores debería ser simple, esa es la idea con ArgenMap. Para crear un marcador se puede suministrar un objeto con opciones o simplemente hacer la llamada $(selector).agregarMarcador() y por defecto el marcador se agregará en el centro del mapa.

Para personalizar el comportamiento del marcador pueden utilizarse estas opciones

Opcionales
 
nombre
cadena de texto indicando el nombre que tendrá el marcador. Recordá que si bien es opcional, es importante definir el nombre e intentar que sea único para luego poder accederlo si necesitaras modificarlo.
lat
Las coordenadas pueden proveerse por separado, nombrando cada valor por su eje (lon y lat)
lng
Las coordenadas pueden proveerse por separado, nombrando cada valor por su eje (lat y lng)
contenido
cadena de texto con contenido HTML que se mostrará al hacer click sobre el marcador en un globo de texto.

$(selector).agregarMarcadores([marcador1, marcador2,..., marcadorN])

Crea varios marcadores sobre el mapa a partir de una array de objetos que definen marcadores. Cada objeto es igual al que recibe agregarMarcador()

API versión 1.0

Esta API es completamente funcional pero recomendamos comenzar a utilizar la API 1.2 porque es mucho más práctica, sobre todo para quienes están comenzando a utilizar jQuery.
A partir de enero de 2013 vamos a dejar de documentar la API 1.0 y a dejar de desarrollarla.

Formato de los datos

Simplificamos algunos formatos de la API de Google Maps con argenmap.jquery (pero igual se pueden utilizar los objetos originales de GMaps API).

latlng

Podés usar el objeto google.maps.LatLng pero de manera simplificada:

  • {lat:float, lng:float} : objeto de 2 números decimales
  • [lat, lng] array de 2 números decimales

LatLngBounds

El objeto google.maps.LatLngBounds lo simplificamos así:

  • {ne:LatLng, sw:LatLng} : objeto de 2 LatLng
  • [ne, sw] : array de 2 LatLng
  • {n:float, e:float, s:float, w:float} : objeto de 4 números decimales
  • [n, e, s, o] : array de 4 números decimales

Conversión de direcciones a coordenadas

La mayoría de las acciones de argenmap.jquery resuelven direcciones automáticamente por medio del parámetro direccion en lugar de utilizar latLng.

Por ejemplo, la acción agregarMarcador recibe coordenadas (latLng) para posicionar un nuervo marcador. Podés usar una dirección en lugar de sus coordenadas. La dirección se transforma automáticamente a coordenadas.

Tené en cuenta que la conversión de direcciones a coordenadas es asincrónica por el intercambio entre tu browser y el servicio $

direccion

direccion puede ser:

  • string: la dirección como texto
  • GeocoderRequest: un objeto de google completo

Parámetros de argenmap

Para su uso principal, argenmap toma objetos como parámetros. Cada objeto es una acción a ejecutar. Estas acciones son ejecutadas según su orden en la lista de parámetros, una por vez, cuando la acción anterior haya terminado.

Este ejemplo primero inicializa el mapa, y luego agrega dos marcadores. 

$('#ejemplo').argenmap({
	accion:'init',
	opciones: {
		center:[-34, -59],
		zoom: 5
	}
}, {
	accion: 'agregarMarcador',
	latLng:[-34, -59]
}, { 
	accion: 'agregarMarcador',
	latLng:[-34.2, -59.2]
});

El objeto de acción usa estos parámetros

  • accionstringnombre de la función a utilizar
  • opcionesobjetoopciones a aplicar al objeto que se está creando (marcador, overlay ...)

Con algunas acciones específicas se puede incrustar otros parámetros que no están en esta lista

Acciones

El valor de accion puede ser

  • agregarMarcadorcrea y agrega un google.maps.Marker
  • agregarMarcadorescrea y agrega varios google.maps.Marker
  • initInicia el mapa
  • limpiarLimpia los objetos creados sobre el mapa

Parámetro callback

Algunas acciones, aceptan un parámetro callbacken el objeto de parámetros porque son acciones que se ejecutan asincrónicamente. Las funciones de callback reciben un parámetro que es el resultado de la acción ejecutada y depende de la acción.

  • agregarMarcador manda un google.maps.Marker
  • agregarCirculo manda un google.maps.Circle
  • geoLatLng manda un google.maps.LatLng
  • ...

En este ejemplo podés ver dos cosas importantes de argenmap:
en la función de callback $(this) equivale a $('#mapa').
setCenter es una llamada a la función de google.maps.Map (setCenter no es una función de argenmap), por eso el argumento a esta función está en la array args.

$('#mapa').argenmap({
	accion : 'geoLatLng',
	callback : function(latLng) {
		if (latLng) {
			$(this).argenmap({accion: 'setCenter', args:[ latLng ]});
		} else {
			alert('No se pudo obtener la geolocalización');
		}
	}
});

Eventos

Los eventos que quieras enganchar a un mapa se guardan en eventos y unicos. Todos usan este prototipo: function (origen , evento , data, )

  • origen depende de la acción (agregarMarcador manda un google.maps.Marker, ...)
  • evento es el evento javascript q disparó esta función
  • data es lo que el usuario ingresa y puede ser cualquier cosa (string, array, objeto, booleano, ...)

En este ejemplo, usamos una string como data para cada marcador y se muestra cuando el evento click se dispara en estos marcadores

$('#mapa').argenmap({
	accion:'init',
	opciones: {
		center:[-34,-59],
		zoom: 5
	}
}, {
	accion: 'agregarMarcadores',
	markers: [
		{lat: -25.695278, lng: -54.436667, data:'Cataratas del Iguazú!'},
		{lat: -34.05, lng: -59.2, data:'Atucha: central eléctrica'},
		{lat: -27.31,  lng: -66.608617, data:'Minera Lumbrera: buena mina'}
	],
	marker: {
		opciones: {
			draggable: false
		},
		events: {
			click: function(marker, event, data) {
				alert(data);
			}
		}
	}
});
